NOTES ON CURVE NAMING & RELATIONSHIPS
=====================================

Summary:
- Curve25519     → A Montgomery elliptic curve
- Edwards25519   → A twisted Edwards elliptic curve
                   (also called Ed25519 curve)
- X25519         → A key exchange algorithm that uses Curve25519
- Ed25519        → A digital signature algorithm that uses Edwards25519

Important Point:
Curve25519 and Edwards25519 are NOT two completely different curves.
They are two different representations (forms) of essentially the same
curve. They are birationally equivalent — meaning there is a mathematical
mapping that converts points between them (defined in RFC 7748).

Design Decision:
We use Ed25519 as the primary identity and signing curve.
X25519 keys are derived on-the-fly for key exchange using the
birational mapping (RFC 7748).

# 5. Public Key Validation

FUNCTION ValidateX25519PublicKey(
publicKey : BYTE_ARRAY[32]
)

RETURNS:
isValid : BOOLEAN

DESCRIPTION:
Validates that a public key is safe for X25519 key exchange.

REQUIRED CHECKS:

```
1. Length:
    - MUST be exactly 32 bytes  

2. Non-zero:
    - MUST NOT be all-zero  

3. Low-order points:
    - MUST reject known low-order points for Curve25519  
```

RETURN:
TRUE  → valid
FALSE → reject


# 15. High-Level Operations

## 15.1 Create Envelope

FUNCTION CreateEnvelope(
plaintext           : BYTE_ARRAY,
senderPrivateKey    : BYTE_ARRAY[32],
recipientPublicKey  : BYTE_ARRAY[32]
)

RETURNS:
envelope : Envelope

STEPS:

```
senderPublicKey = DerivePublicKey(senderPrivateKey)  

senderXPriv   = ConvertEd25519PrivateKeyToX25519(senderPrivateKey)  
recipientXPub = ConvertEd25519PublicKeyToX25519(recipientPublicKey)  

sharedSecret  = DeriveSharedSecretSafe(senderXPriv, recipientXPub)  

aesKey        = DeriveAES256Key(sharedSecret)  

nonce         = GenerateNonce()  

ciphertext, tag = Encrypt(plaintext, aesKey, nonce)  
```

---

## 15.2 Open Envelope

FUNCTION OpenEnvelope(
envelope            : Envelope,
recipientPrivateKey : BYTE_ARRAY[32]
)

RETURNS:
plaintext : BYTE_ARRAY

ERROR:
AuthenticationFailure
InvalidPublicKey

STEPS:

```
recipientXPriv = ConvertEd25519PrivateKeyToX25519(recipientPrivateKey)  
senderXPub     = ConvertEd25519PublicKeyToX25519(envelope.senderPublicKey)  

sharedSecret   = DeriveSharedSecretSafe(recipientXPriv, senderXPub)  

aesKey         = DeriveAES256Key(sharedSecret)  

plaintext      = Decrypt(
                    envelope.ciphertext,
                    envelope.tag,
                    aesKey,
                    envelope.nonce
                 )  
```

# Hard Requirements

1. All fixed-size fields MUST match exact sizes
2. Nonce MUST NEVER repeat for same AES key
3. DeriveSharedSecretSafe MUST be used (never raw DeriveSharedSecret)
4. AES-GCM tag MUST be verified before returning plaintext
5. Serialization MUST follow canonical format exactly
6. No implicit encoding assumptions (no strings)
7. Reject invalid or malformed public keys

Federated encrypted messaging system (1 request = 1 envelope)

#include <stdint.h>

uint8_t is an unsigned 8-bit integer type defined in <stdint.h>

/* ---------------------------------------------------------
   1. CONTENT TYPE
--------------------------------------------------------- */

typedef uint8_t ContentType;

#define CT_TEXT_UTF8  0x01
#define CT_IMAGE      0x02
#define CT_VIDEO      0x03
#define CT_AUDIO      0x04
#define CT_FILE       0x05
#define CT_UNKNOWN    0xFF


/* ---------------------------------------------------------
   1. ENVELOPE STRUCTURE (ON WIRE FORMAT)
--------------------------------------------------------- */

typedef struct Envelope
{
    /* 32-byte Ed25519 public keys */
    uint8_t senderPublicKey[32];
    uint8_t recipientPublicKey[32];

    /*
    32-byte nonce
    MUST be unique per (sender, recipient) pair

    message_id = SHA-256(sender || recipient || nonce)
    */
    uint8_t nonce[32];

    /* Client-provided UTC timestamp (seconds since epoch) */
    uint64_t timestamp;

    /* Content type hint (plaintext interpretation only) */
    uint8_t contentType;

    /* AES-GCM parameters */
    uint8_t aesNonce[12];
    uint8_t authTag[16];

    /* Variable-length ciphertext */
    uint8_t ciphertext[];
} Envelope;


/* ---------------------------------------------------------
   2. MESSAGE ID (DEDUPLICATION KEY)
--------------------------------------------------------- */

/*
Computed by relay:

message_id = SHA-256(
    senderPublicKey ||
    recipientPublicKey ||
    nonce
);
*/


/* ---------------------------------------------------------
   3. POST /envelope
--------------------------------------------------------- */

POST /envelope?

Request body:
    Envelope (raw binary)

Server behavior:

- Validate:
    senderPublicKey size = 32
    recipientPublicKey size = 32
    nonce size = 32
    timestamp within allowed bounds:
        (now - MAX_AGE) <= timestamp <= (now + 300)

- Compute message_id
- If message_id exists → ignore (idempotent)
- Store envelope
- If forward=1 → async send to federated peers

Response:
    200 OK


/* ---------------------------------------------------------
   4. GET /envelope
--------------------------------------------------------- */

GET /envelope?pubkey=<hex>&cursor=<uint64 optional>

Behavior:

- cursor default = 0
- find envelope where:
      recipientPublicKey == pubkey
      AND timestamp > cursor

- return OLDEST matching envelope

Response:

If found:
    Content-Type: application/octet-stream
    Body: Envelope (raw binary)

If none:
    204 No Content


/* ---------------------------------------------------------
   5. RULES
--------------------------------------------------------- */

/*
- Server NEVER decrypts ciphertext
- Server NEVER interprets contentType
- Server NEVER trusts timestamp beyond sanity check
- nonce MUST be unique per sender-recipient pair
- message_id is deterministic and used for deduplication
- Envelope format is identical across all relays
*/
